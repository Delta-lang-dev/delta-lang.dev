migrate

Upgrade from one API version to another across your codebase.

migrate handles the mechanical part of API migrations: renaming symbols, moving files, removing deprecated functions. Write the migration rules once and apply them project-wide.

Syntax

migrate "label" in Language {
  rename: oldName -> newName
  move:   "old/path.ts" -> "new/path.ts"
  remove: deprecated symbolName
  scope:  "glob/**"  // optional
}

Example

migrate "lodash 4 to native" in TypeScript {
  rename: _.map    -> [].map
  rename: _.filter -> [].filter
  rename: _.reduce -> [].reduce
  rename: _.find   -> [].find
  remove: deprecated _.each
  scope:  "src/**/*.ts"
}

apply migrate "lodash 4 to native" to project preview

Fields

FieldRequiredDescription
label✓Unique label — referenced by apply
in Language✓Target language
rename: A -> B✓Rename a symbol everywhere it appears
move: A -> B✓Move a file and update all import paths
remove: deprecated X✓Remove calls to a deprecated symbol
scope✓Glob to limit which files are affected

Tips

  • Chain multiple rename rules in one migrate block
  • move: updates import paths in all other files automatically
  • Always preview a migration first — it affects many files
  • Pair with guard to assert the new API exists before migrating to it
