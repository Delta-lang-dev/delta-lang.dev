guard

Assert invariants before any transform runs.

guard checks conditions in your codebase before allowing transforms to proceed. If a guard fails, the apply is blocked and a diff is shown. Use guards to prevent transforms from running in an inconsistent state.

Syntax

guard "label" {
  scope:   "glob/**"
  assert:  fn name() exists
  assert:  no file imports from "path"
  assert:  all files contain pattern
  on_fail: block_apply | warn | show_diff
}

Example

guard "auth-api-contract" {
  scope:   "src/**"
  assert:  fn verifyToken() exists
  assert:  fn hashPassword() exists
  assert:  no file imports from "../admin"
  on_fail: block_apply, show_diff
}

guard "no-raw-sql" {
  scope:   "src/**/*.ts"
  assert:  no raw f-string SQL queries exist
  on_fail: block_apply, warn
}

Fields

FieldRequiredDescription
label✓Unique label for this guard
scope✓Files to check
assert: fn name() exists✓Passes if the named function exists anywhere in scope
assert: no file imports from✓Passes if no file imports from the given path
assert: all files contain✓Passes if every file in scope contains the pattern
on_fail✓block_apply, warn, show_diff — can combine with comma

Tips

  • Guards run before patches, fixes, and migrations — always
  • Combine multiple asserts in one guard block
  • on_fail: block_apply is the safest default for production codebases
  • Use guard + trace together for security auditing workflows
