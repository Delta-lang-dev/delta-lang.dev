trace

Follow data through your codebase and flag dangerous paths.

trace performs data-flow analysis. Give it a variable or expression as origin and it follows every path that value can take — through assignments, function calls, and returns — reporting any path that matches your flag conditions.

Syntax

trace variableName in "directory/" {
  language: Language
  origin:   source.expression
  follow:   assignments, function_calls, returns
  report:   full_chain | summary
  flag:     "condition to flag"
}

Example

trace userInput in "src/" {
  language: TypeScript
  origin:   req.body.email
  follow:   assignments, function_calls, returns
  report:   full_chain
  flag:     "any path that reaches db.query without sanitize"
  flag:     "any path that reaches console.log"
  flag:     "any path that reaches res.send without encode"
}

Fields

FieldRequiredDescription
ident✓Name for this trace (used in report output)
in directory✓Directory to analyse
language✓Language to use for AST parsing
origin✓Starting expression (e.g. req.body.field)
follow✓What to follow: assignments, function_calls, returns
report✓full_chain (every step) or summary (flagged paths only)
flag✓Natural-language condition — flagged in output if matched

Tips

  • trace is Delta's security construct — use it to audit user input handling
  • Combine with guard and intent for a full security hardening workflow
  • Multiple flag conditions can be specified in one trace block
  • report: summary is faster for large codebases — only shows flagged paths
